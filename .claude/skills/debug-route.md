# Skill: Debug Route Issues

Use this skill when diagnosing routing or redirect problems.

## Common Issues

### 1. Trailing Slash Redirects
The root loader calls `removeTrailingSlashes(request)` which redirects URLs with trailing slashes. If this gets accidentally removed (as happened during the RR7 migration), trailing slash URLs may 404 or cause inconsistent behavior.

**Check:** Verify `removeTrailingSlashes(request)` is called in `app/root.tsx` loader.

### 2. Express Static Conflicts
Journal images are served at `/build/journal` (maps to `content/journal/`). If this static mount conflicts with a route path, `express.static` may redirect directory requests with trailing slashes, causing loops.

**Check:** Ensure no route path matches a static mount path in `server.js`.

### 3. Route Type Errors
Route types are generated by `npx react-router typegen`. If types are stale or missing:
```sh
npx react-router typegen
npx tsc --noEmit
```

### 4. Module Resolution Collision
Files at the project root named after npm packages (e.g., `react-router.d.ts`) will shadow the package due to `baseUrl: "."` in tsconfig. Module augmentation files should go in `types/`.

### 5. Error Boundaries
Use `ErrorBoundary` with `useRouteError()` and `isRouteErrorResponse()`. Never use `CatchBoundary` (Remix v1 API, dead code in React Router v7).

Throw `new Response(message, { status })` from loaders/actions for proper error handling:
```tsx
throw new Response('Not Found', { status: 404 })
```

## Debugging Steps

1. Check `server.js` middleware order and static mounts
2. Check `app/routes.ts` for route path conflicts
3. Check `app/root.tsx` loader for redirect logic
4. Run `npx react-router typegen && npx tsc --noEmit` for type issues
5. Check `fly.toml` for proxy-level redirects (force_https)
6. Check browser Network tab for redirect chains
